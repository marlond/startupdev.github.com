FAQ

O que é o Startup DEV RUMBLE?

É uma competição de programação onde equipes de até 4 pessoas terão 48 horas para construir uma inovadora aplicação web na linguagem e framework que desejar. Após as 48 horas a aplicação será julgada pela equipe do Startup DEV RUMBLE, em o que foi entregue, interface do usuário, originalidade e utilidade.

Quando inicia-se a competição?

A competição inicia-se ás 00:00 de 01 de setembro de 2012 e continua nas próximas 48 horas terminando ás 23:59 de 02 de setembro de 2012.

Onde será realizado o concurso?

Virtualmente, isto quer dizer que seu time pode trabalhar de qualquer lugar. Não é o futuro a internet?

Quando é aberto o cadastro?

O cadastro será aberto a partir de primeiro de agosto de 2012. Fique atento no twitter da HE:labs para as novidades e qualquer dúvida entre em contato conosco através do email contato@helabs.com.br. Tentaremos abrir a maior quantidade de vagas possiveis mais teremos um espaço limitado, então lembre-se que os primeiros terão prioridade.

O que eu preciso saber para poder entrar na competição?

Conhecimento em alguma linguagem de programação que rode na plataforma Heroku e HTML seria um bom começo. Também deve ter conhecimento básico em Git (para controle de código). Nós iremos providenciar a você um repositório Git e uma app no heroku.

Onde será hospedado os repositórios Git?

O codeplane irá providenciar a cada equipe um repositório privado grátis, durante a competição. Você deve usar este repositório para o desenvolvimento (é uma grande desculpara para aprender Git se você ainda não sabe!) e alguém do seu time deve commitar pelo menos 2 vezes por dia para demonstrar progresso.

Onde a aplicação será hospedada?

Cada equipe terá uma app hospedada no heroku na stack Cedar. Não será necessário montar todo o ambiente de deploy, apenas integrar seu código ao heroku que ele cuida do ambiente para você.

Se minha empresa estiver patrocinando o Startup DEV RUMBLE, eu posso participar da competição?

Com certeza!

Em qual sistema operacional a aplicação rodará?

O heroku no stack Cedar, usado por todos os participantes, usa o sistema operacional Ubuntu 10.04.

Eu posso usar gems thirdy party, plugins, libraries, templates, icones etc.?

Nós encorajamos o uso de material thirdy party que é publico a todos.

Posso desenvolver um plugin ou uma gem para minha aplicação antes da competição iniciar?

Desde que o plugin esteja disponível ao público e não fornece a maioria das funções do aplicativo. Seria aceitável para desenvolver e lançar um plugin que fornece uma interface para uma API ou construir formulários, por exemplo. Seria inaceitável esconder seus controllers e views dentro de um plugin, "lança-lo", e instalá-lo uma vez que a competição começa.

O que estou autorizado a fazer para se preparar para a competição?

Você é livre para planejar e projetar os requisitos para sua aplicação. Você também é livre para trabalhar em papel e/ou mockups da interface do usuário. Você não tem permissão para produzir os nenehum material digital ou qualquer teste ou código do aplicativo antes do início da competição. Se você tiver dúvidas, não hesite em contatar os organizadores.

Quando eu terei acesso ao repositório de código e a app no heroku?

O acesso a ambos será liberado no inicio da competição.

A minha aplicação precisa ser open source?

Enquanto incentivamos as equipes a compartilhar seu código fonte, a competição não exige isso. Esperamos que alguns de vocês vão entregar propriedades da web verdadeiramente inovadores durante o evento, não é uma exigência de que você compartilha o seu código com os outros a menos que queira.

Onde eu posso encontrar as minhas informações da conta no codeplane e no heroku?

Ao finalizar a inscrição lhe enviaremos todas as informações que você possa precisar.

Posso dar push de algumas coisas antes do inicio da competição?

NÃO.

Posso mudar o nome do meu repositório?

NÃO.

Quem faz este concurso acontecer?

O Startup DEV RUMBLE é o trabalho duro da equipe da HE:labs que faz o Startup DEV que foi de onde a ideia do Startup DEV RUMBLE surgiu. A equipe escreve o código para realizar o evento e coordena o que precisa de coordenação.

Apoio significante também é doado por nossos parceiros de infra-estrutura o Codeplane e o Heroku, sem os quais este simplesmente não seria possível.
