FAQ

O que é o Startup DEV RUMBLE?

É uma competição de programação onde equipes de até 4 pessoas terão 48 horas para construir uma inovadora aplicação web na linguagem e framework que desejar.

Onde será realizado o concurso?

Virtualmente. Isto quer dizer que seu time pode trabalhar de qualquer lugar. Não é o futuro a internet?

Quando serão abertas as inscrições?

As datas estão divulgadas na página inicial da competição. Fique atento no twitter da HE:labs para as novidades e qualquer dúvida entre em contato conosco através do email contato@helabs.com.br.

Há limitação de vagas?

Tentaremos abrir a maior quantidade de vagas possíveis, mas teremos um espaço limitado, então lembre-se que os primeiros terão prioridade.

O que eu preciso saber para poder entrar na competição?

Conhecimento em alguma linguagem de programação que rode na plataforma Heroku, HTML e javascript seria um bom começo. Também deve ter conhecimento em Git, para controle de versão do código.

Onde serão hospedados os repositórios Git?

O codeplane irá providenciar a cada equipe um repositório privado grátis durante a competição. Você deve usar este repositório para o desenvolvimento (é uma grande desculpa para aprender Git se você ainda não sabe!) e alguém do seu time deve integrar pelo menos 2 vezes por dia para demonstrar progresso.

Onde a aplicação será hospedada?

Cada equipe terá uma app hospedada no Heroku na stack Cedar. Não será necessário montar todo o ambiente de produção, mas apenas colocar seu código no Heroku que ele cuida do ambiente para você.

Se minha empresa estiver patrocinando o Startup DEV RUMBLE, eu posso participar da competição?

Com certeza!

Em qual sistema operacional a aplicação rodará?

O Heroku no stack Cedar, usado por todos os participantes, usa o sistema operacional Ubuntu 10.04.

Eu posso usar gems, plugins, bibliotecas, templates, ícones, etc?

Sim. Nós encorajamos o uso de qualquer material que seja público a todos.

Posso desenvolver um plugin ou uma biblioteca para minha aplicação antes da competição iniciar?

Desde que o plugin ou biblioteca esteja disponível ao público e não forneca a maioria das funções do aplicativo. Seria aceitável para desenvolver e lançar um plugin que fornece uma interface para uma API ou construir formulários, por exemplo. Seria inaceitável esconder seus controllers e views dentro de um plugin e "lançá-lo" apenas para uso na competição.

O que estou autorizado a fazer para me preparar para a competição?

Você é livre para planejar e projetar os requisitos para sua aplicação. Você também é livre para trabalhar em papel e/ou mockups da interface do usuário. Você não tem permissão para produzir material digital ou qualquer teste ou código do aplicativo antes do início da competição. Se você tiver dúvidas, não hesite em nos contatar.

Quando eu terei acesso ao repositório de código e a app no Heroku?

O acesso a ambos será liberado no inicio da competição.

A minha aplicação precisa ser open source?

Incentivamos as equipes a compartilhar seu código fonte, mas a competição não exige isso.

Posso dar push de algumas coisas antes do inicio da competição?

NÃO.

Posso mudar o nome do meu repositório?

NÃO.

Como posso obter mais informações?

Fique atento no twitter da HE:labs para as novidades e qualquer dúvida entre em contato conosco através do email contato@helabs.com.br.

Minha empresa ou organização gostaria de patrocinar o evento. Como posso fazer isso?

Entre em contato com os organizadores do concurso através do email contato@helabs.com.br.

Quem devo contatar para assessoria de impresa?

Fale conosco através do email contato@helabs.com.br.
